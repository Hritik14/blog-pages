Web cheatsheet

2025-06-28

You can send postMessage to any window, even if it’s a different origin.

User input in document.querySelector can enable to select malicious item using nth-child

SOP does not block access to window.frames.length (to do postMessage) to a different origin.

Blob documents (data:text/html, …)created from null origins are run in isolated process for security [ONLY IN CHROME]

== converts type to match. ie e.data == 'blob loaded' will take long time if e.data is not string and conversion is costly

window[0] is same as window.frames[0] is same as document.querySelectorAll('iframe')[0].contentWindow

origin and location.href can be very different. Eg: sandboxed iframe has null origin but location.href is correct url

Prototype pollution⌗

  1. __proto__ is same as constructor.prototype

Regex⌗

  1. /token/g.test(target) cannot be used multiple times on different targets. It maintains lastIndex of last match and restarts from there. ie if target="token", it will match first time, then set index to 5 and start match from there, matched again against same token, it will return false
