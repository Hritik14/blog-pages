Docker mount shenanigans

2025-09-03

Works⌗

Problem:

Want to get everything under xx8: in immich as external library BUT immich data dir is also under xx8 ie xx8/immich don’t want to get that data dir as external library as it contains thumbnails already

Best possible way: External lib -> Add exclusion pattern -> /home/user/drive/xx8/immich/**

services: immich-server: container_name: immich_server image: ghcr.io/immich-app/immich-server:${IMMICH_VERSION:-release} # extends: # file: hwaccel.transcoding.yml # service: cpu # set to one of [nvenc, quicksync, rkmpp, vaapi, vaapi-wsl] for accelerated transcoding volumes: - ./mounts/xx8/immich:/data:rw - ./mounts/:/home/user/drive:ro - /home/user/drive/xx8/immich # ensure generated thumbnails are not indexed again - ${THUMB_LOCATION}:/data/thumbs - /etc/localtime:/etc/localtime:ro

dietpi@DietPi:~/immich$ sudo docker compose exec immich-server bash root@42579023a9d8:/usr/src/app# mount | grep xx8 xx8: on /data type fuse.rclone (rw,nosuid,nodev,relatime,user_id=1000,group_id=1000,allow_other) xx8: on /home/user/drive/xx8 type fuse.rclone (ro,nosuid,nodev,relatime,user_id=1000,group_id=1000,allow_other) /dev/root on /home/user/drive/xx8/immich type ext4 (rw,noatime,lazytime)

dietpi@DietPi:~/immich$ ls -ld empty/ drwxr-xr-x 2 dietpi dietpi 4096 Sep 1 12:35 empty/

———————— Weirdo ————

Doing either of two works now. Hypothesis: if it’s a FUSE fs and fs is NOT cached yet then

  - ./mounts/:/home/user/drive:ro
  - ./empty:/home/user/drive/xx8/immich

./empty mount does not work and contents for /home/user/drive/xx8/immich are fetched from fuse -> /mounts/xx8/immich

After FUSE fs is cached ie docker knows /mounts/xx8/immich exists, then ./empty mount is used for /home/user/drive/xx8/immich

once i did services: immich-server: container_name: immich_server image: ghcr.io/immich-app/immich-server:${IMMICH_VERSION:-release} # extends: # file: hwaccel.transcoding.yml # service: cpu # set to one of [nvenc, quicksync, rkmpp, vaapi, vaapi-wsl] for accelerated transcoding volumes: - ./mounts/xx8/immich:/data:rw - ./mounts/:/home/user/drive:ro - ./empty:/home/user/drive/xx8/immich

then i did services: immich-server: container_name: immich_server image: ghcr.io/immich-app/immich-server:${IMMICH_VERSION:-release} # extends: # file: hwaccel.transcoding.yml # service: cpu # set to one of [nvenc, quicksync, rkmpp, vaapi, vaapi-wsl] for accelerated transcoding volumes: - ./mounts/xx8/immich:/data:rw - ./mounts/:/home/user/drive:ro - /home/user/drive/xx8/immich

but /home/user/drive/xx8/immich still points to empty, why

doing thsi solved it docker compose down -v –remove-orphans why ?

dietpi@DietPi:/immich$ sudo docker compose down -v [+] Running 6/6 ✔ Container immich_machine_learning Removed 0.0s ✔ Container immich_server Removed 0.0s ✔ Container immich_postgres Removed 0.0s ✔ Container immich_redis Removed 0.0s ✔ Volume immich_model-cache Removed 0.0s ✔ Network immich_default Removed 0.3s dietpi@DietPi:/immich$
